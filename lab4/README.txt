У нас есть верстка, в ней есть форма, у нее должна быть кнопка submit - Когда мы её нажимаем - браузер автоматом
берёт всё что есть в форме и формирует тип FormData, это фактические ассоциативный массив. В нём ключом явяется
name и есть какое-то значение. <name>; <value>   . Браузер передает всё это через Body\

Заголовок:
	URL: http://localhost
	QuerryString: 

Body существут тогда, когда POST, PUT...
GET - имеет ограничение на длину строки и используется для того, чтобы что-то получить. Но длина QuerryString 255
POST - нужен для того чтобы отправлять какие-то данные, в нём используется body, он может быть большим.

Протокол http состоит из заголовка и тела, внутри тела что-угодно, а в заголовке должна быть метаинфа

Внутри body может быть что угодно, его, в отличии от FormData может передаваться что-угодно
